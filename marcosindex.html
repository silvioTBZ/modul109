Auftrag 1.2: Grundlagen Cloud-Computing

1. Wie ist der Begriff Cloud entstanden? Wieso heisst es Cloud?
Man wollte mit Cloud die Art und Weise beschreiben wie IT-Dienste und Ressourcen über das Internet bereitgestellt und genutzt wird.

2. Wie wird der Begriff Cloud definiert, z.B. gemäss NIST?
Gemäss NIST(National Institute of Standards and Technology) wird Cloud als, ein Modell zur Ermöglichung eines allgegenwärtigen, bequemen On-Demand-Netzwerkzugriffs auf einen gemeinsamen Pool konfigurierbarer Computerressourcen, definiert.

3. Welches sind die 5 Merkmale einer Cloud?
Sofort verfügbar, Ortsunabhängig, Verteilte Ressourcen, Elastisch, man zahlt nur so viel wie man braucht

4. Welche Cloud Dienstleistungen kennen Sie?
Dateiablagen, Datenbanken, Webseiten

5. Welche Cloud Anbieter kennen Sie?
AWS, Azure

6. Welche Cloud Deployment Modelle kennen Sie?
Public Cloud, Private Cloud

7. Was sind Cloud Service Modelle?
Infrastructure as a Service, Software-as-a-Service, Platform-as-a-Service

Die wichtigsten Git Befehle

1. Git clone: Erstellt eine Kopie von einem Remote-Repositories.

2. Git init: Erstellt ein neues Git-Repositories.

3. Git add: Fügt eine Datei zu dem nächsten Commit hinzu

4. Git commit: Erfasst eine äderung

5. Git status: Zeigt den Status von dem Arbeitsverzeichnis an

6. Git push: Übertragt die aktuellen Commits endgültig in das Remote-Repositories

7. Git pull: Aktualisiert den lokalen Branch mit den neuesten Änderungen aus dem Remote-Repository

8. Git branch: Zeigt eine Liste der vorhandenen Branches an

9. Git log: Zeigt die Commit-Historie
